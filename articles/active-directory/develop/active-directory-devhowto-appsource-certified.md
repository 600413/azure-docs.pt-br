# <a name="how-to-get-appsource-certified-for-azure-active-directory-ad"></a>Como certificar o AppSource para o Azure AD (Active Directory)
Para receber a certificação do AppSource para o Azure AD, seu aplicativo deve implementar padrão de entrada multilocatário com o Azure AD usando os protocolos OpenID Connect, OAuth 2.0 ou SAML 2.0. 

Se você não estiver familiarizado com o desenvolvimento de aplicativos multilocatário ou de entrada no Azure AD:

1. Comece lendo sobre o [Navegador para cenários de aplicativo Web em cenários de autenticação do Azure AD][AAD-Auth-Scenarios-Browser-To-WebApp].  
2. Em seguida, confira os [guias de início rápido do aplicativo Web][AAD-QuickStart-Web-Apps] do Azure AD, que demonstram como implementar a entrada e incluem exemplos de código complementar. 
   
   > [!TIP]
   > Experimente a demonstração do nosso novo [portal do desenvolvedor](https://identity.microsoft.com/Docs/Web) que ajudará você a executar o Azure Active Directory em apenas alguns minutos!  O portal do desenvolvedor orientará você pelo processo de registro de um aplicativo e integração do Azure AD em seu código.  Quando terminar, você terá um aplicativo simples que pode autenticar os usuários em seu locatário e um back-end que pode aceitar tokens e executar a validação.
   > 
   > 
3. Para saber como implementar o padrão de entrada multilocatário com o Azure AD, confira [Como entrar com qualquer usuário do Azure AD (Active Directory) usando o padrão de aplicativo multilocatário][AAD-Howto-Multitenant-Overview]
